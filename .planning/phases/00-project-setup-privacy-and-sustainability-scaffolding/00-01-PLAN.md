<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External contributor → repo | Untrusted contributor opens PR; CI is the gate before any code merges |
| GitHub web UI → git history | Maintainer using "Squash and merge" UI can edit messages and drop DCO trailers |
| npm registry → repo dependencies | Future deps may carry incompatible licenses (AGPL/SSPL etc.) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-01 | Tampering / Information Disclosure | npm transitive dependency tree | mitigate | `.github/workflows/license-check.yml` runs `pnpm licenses list --json` on every PR with extended denylist (GPL-2.0, GPL-3.0, AGPL-1.0, AGPL-3.0, LGPL-2.1, LGPL-3.0, SSPL-1.0, RSAL-2.0, RSALv2, BUSL-1.1, ELv2, Confluent-Community-1.0, MariaDB-BSL-1.1) and uses `--ignore-scripts` to prevent malicious postinstall execution during license check |
| T-00-02 | Repudiation / Tampering | PR commits | mitigate | `.github/workflows/dco.yml` requires `Signed-off-by:` trailer on every commit via `KineticCafe/actions-dco@v2` with `fetch-depth: 0` for full history scan |
| T-00-03 | Repudiation | Squash-merge UI dropping sign-off trailers | mitigate | Task 3 (maintainer-action checkpoint) enables GitHub setting "Require contributors to sign off on web-based commits"; DCO workflow gates PR commits not merge commits, so PR-time DCO compliance is preserved even if maintainer edits squash message |
| T-00-04 | Information Disclosure | Secrets in `.env`, `*.key`, `*.pem` | mitigate | `.gitignore` includes these patterns; only `.env.example` is committed (no real secrets in repo) |
| T-00-05 | Tampering | License-check CI being silently skipped on private repo (T-supply-chain-license variant) | mitigate | Use `pnpm licenses list --json` (works on any repo) NOT `actions/dependency-review-action` (requires GHAS on private repos per Pitfall 1); add fixture PR test in Plan 02 verify-phase-0.sh to confirm denylist actually rejects an AGPL package |
| T-00-06 | Repudiation | Contributor without right to contribute (T-contributor-provenance) | mitigate | DCO sign-off via `KineticCafe/actions-dco@v2`; CONTRIBUTING.md documents inbound=outbound Apache-2.0; explicit "no CLA" policy makes the IP origination trail clear |
</threat_model>
